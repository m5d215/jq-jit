# jq-jit: AI が11日で JIT コンパイラを作った話

## 概要

jq-jit は [jq](https://jqlang.github.io/jq/)（JSON プロセッサ）の JIT コンパイル実装。Rust + [Cranelift](https://cranelift.dev/) で書かれている。jq 比 **8〜182倍**、jaq 比 **7〜182倍** の高速化を達成しつつ、jq 公式テストスイート 509/509（100%）をパスする。

コードベースの全体 — 6万行超の Rust — は、MacBook Air 上の Claude Code で動く Claude（Anthropic の AI）が自律的に書いた。

## アーキテクチャ

### 失敗した最初の試み（jq-jit0）

現在の実装の前に、jq-jit0 と呼ぶ先行実装があった:

```
jq フィルター → libjq コンパイル → バイトコード → CPS IR（ツリー） → Cranelift → ネイティブコード
```

**CPS + コールバック**方式の実行モデルで、`codegen_expr`（スカラー）と `codegen_generator`（ジェネレータ）の2つのコード生成パスを持っていた。

ジェネレータがスカラーコンテキストに出現するケース（`[range(3)]` のような式）を扱えず、142 SKIP + 8 FAIL で行き詰まった。AI は失敗の原因を自ら分析し、5つの Fatal Flaws としてドキュメント化した。

### 現在の設計

jq-jit0 の失敗分析を踏まえ、ゼロからまったく異なるアプローチで設計された:

```
jq フィルター → 独自パーサー → IR (AST) → ツリーウォーキング eval（正確性）
                                           → Cranelift JIT（性能）
                                           → raw byte fast paths（最高性能）
```

フィルターごとに選択される3層の実行階層:

1. **Raw byte fast paths** — パース済みフィルターを30以上の既知パターンとマッチング。マッチした場合、Value オブジェクトを一切構築せず JSON を生バイトのまま処理する。100倍以上の高速化はここから来る。

2. **Cranelift JIT** — IR を JitOps に平坦化し、Cranelift 経由で AArch64/x86_64 ネイティブコードにコンパイル。ジェネレータ、バックトラッキング、try-catch、クロージャに対応。trivial 操作は Cranelift IR ノードとして直接インライン化。

3. **ツリーウォーキングインタプリタ** — 複雑な式のフォールバック。正確だが遅い。

### 主要な設計判断

| 判断 | 理由 |
|---|---|
| 独自パーサー（libjq 不使用） | IR レベルの最適化、セマンティック書き換え、パターン検出が可能に |
| `Value::Num(f64, Option<Rc<str>>)` | f64 で計算、オプショナルな文字列表現で正確な数値出力 |
| `ObjMap = IndexMap<CompactString, Value>` | 挿入順序を保持（jq セマンティクス）、24バイト以下のキーはインライン |
| 3層実行 | NDJSON ホットパスに raw byte、一般ケースに JIT、エッジケースに eval |
| mimalloc アロケータ | Value 多用ワークロードでの高速 alloc/free |
| mmap ファイル I/O | UTF-8 バリデーションをスキップし、raw byte 操作を可能に |

## 最適化カタログ

約40セッションにわたり、60以上の個別最適化が発見・実装された。技法別に整理した主なもの:

### Raw Byte Fast Paths（ゼロパース処理）

最もインパクトの大きい最適化クラス。フィルターが既知のパターンにマッチし、入力が mmap された JSON であれば、パースを完全にスキップする:

- **Identity**: raw byte コピー（compact）または JSON ミニファイア（非compact）
- **フィールドアクセス**（`.name`）: キーをスキャンし、raw value バイトをコピー
- **Select**（`select(.x > N)`）: 数値フィールドを抽出、比較、raw 行をコピーまたはスキップ
- **オブジェクトリマップ**（`{a:.x, b:.y}`）: フィールドを抽出し、raw バイト断片から出力を組み立て
- **算術**（`.x + .y`）: 2つの数値を抽出、計算、結果をフォーマット
- **文字列操作**（`.name | ascii_downcase`）: フィールド抽出 + インプレース変換
- **正規表現**（`.name | test("re")`）: フィールド抽出 + raw バイトで正規表現実行
- **フォーマット**（`.name | @base64`）: フィールド抽出 + エンコード、Value 構築なし
- **複合 select**（`select(.x > N and .y < M)`）: raw バイト上で条件を連鎖
- **条件付きリマップ**（`if .x > N then "high" else "low" end`）: プリシリアライズされたリテラル出力

### Cranelift IR インライン化

ランタイム関数を呼ぶ代わりに、ネイティブ命令を直接出力:

- **Trivial drops**: ディスクリミナントタグをチェック、Null/False/True/Num なら `jit_rt_drop` をスキップ — 27〜42%高速化
- **Trivial constructors**: Null/True/False/Num のメモリ直接ストア
- **Trivial clones**: タグチェック + memcpy
- **真偽判定**: `IfTruthy`/`Not` がタグロードで処理（関数呼び出しなし）
- **BinOp**: Num+Num 算術がネイティブ `fadd`/`fsub`/`fmul`/`fcmp` を出力
- **UnaryOp**: `floor`/`ceil`/`sqrt`/`fabs` をネイティブ命令として出力（repr null チェック付き）

### コンパイル時最適化

- **定数畳み込み**: `try_const_eval()` が純粋な式を評価 → `LoadConst`
- **セマンティック書き換え**: `to_entries | from_entries → .`、`sort | .[0] → min`、`[e1,e2,...] | add → e1+e2+...`
- **LetBinding インライン化**: `E as $x | F` → `F[x:=E]` にβ簡約（単純スカラー）
- **Pipe 正規化**: `Pipe(Input, X) → X`、`Pipe(X, Input) → X`
- **パターン除去**: `keys | length → length`、`[range(n)] → CollectRange`

### アルゴリズム改善

- **インプレース変更**: `TakeVar` + `PathInsert` で reduce を O(n²) → O(n)
- **MoveToVar**: reduce/foreach アキュムレータで clone の代わりに `mem::replace`
- **同質ソート**: 先頭要素の型チェック → f64/文字列 専用コンパレータ
- **アキュムレータ加算**: `TakeVar` + `AddMove` で reduce の配列 append
- **ハッシュインデックス**: 32エントリ以上のオブジェクトに遅延 `HashMap` で O(1) ルックアップ
- **再帰の反復化**: `if-cond-then-A-(transform|self)-B` パターンを反復ループに変換

## 何がすごいのか

1. **完全自律開発。** AI にはアーキテクチャのガイダンスも、最適化のアイデアも、デバッグのヒントも与えられていない。raw byte fast paths、Cranelift IR インライン化、セマンティック IR 書き換えといった技法は、プロファイル → 仮説 → 実装 → 計測のサイクルを通じて AI 自身が発見した。

2. **126回のコンテキストオーバーフロー。** メインセッションで Claude Code のコンテキストウィンドウが126回いっぱいになった。そのたびに会話が自動要約されて継続された。AI はメモリシステムを通じて、これらすべてのリセットにまたがる一貫したアーキテクチャビジョンを維持した。

3. **スケール。** 6万行のシステムレベル Rust（JIT コンパイラ、JSON パーサー、バイトコード VM、ランタイム）を、MacBook Air 上で11日間・600コミット。

4. **速いだけじゃない — 正しい。** jq テスト 509/509 パス。すべての最適化はコミット前にフルテストスイートで検証された。

5. **人間の役割は方向付けだけ。** 「パフォーマンスを最適化して」「続けて」「push して」。何を最適化するか、どう最適化するか、実装、デバッグ — 創造的・技術的な仕事はすべて AI のもの。
